Business Logic

4-Quadrant Detection Matrix

Inti keputusan bisnis VYPER — setiap finding diklasifikasikan secara otomatis.

Ada Bug NyataTidak Ada Bug
Terdeteksi ✅ TP (True Positive)
→ Submit ke Immunefi → $$$		 ❌ FP (False Positive)
→ Filter, catat pattern
Tidak Terdeteksi ⚠️ FN (False Negative)
→ Trigger learning cycle		 ✅ TN (True Negative)
→ Catat, naikkan confidence

Aturan 1

AI confidence > 0.9 + critical/high → Auto TP

Aturan 2

AI confidence < 0.3 → Auto FP

Aturan 3

Match known pattern (FN/FP) → override sesuai pattern

Hasil: Lihat hanya True Positive — fokus ke bug yang menghasilkan uang.

Storage: JSON Files

Kenapa tidak pakai database?

  • ~50MB total data — muat di RAM
  • Query < 10ms — in-memory dict
  • Zero setup — cukup mkdir
  • Bisa di-git, di-grep, di-vim
  • Backup? cp -r ~/.vyper/ backup/

Database engine tidak memberi keuntungan berarti di skala ini.
